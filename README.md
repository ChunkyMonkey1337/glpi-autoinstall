🚀 GLPI Auto-Installer for Ubuntu

Скрипт для полной автоматизации развертывания системы GLPI (IT Service Management / Helpdesk) на серверах под управлением Ubuntu. Полностью соответствует официальной документации и современным стандартам безопасности.
📌 Что это?

Это bash-скрипт, который за несколько минут превращает "чистую" Ubuntu в готовый к работе сервер GLPI. Скрипт сам установит LAMP-стек, настроит базу данных, скачает нужную версию GLPI, распределит файлы по системным директориям и настроит web-сервер.
🎯 Зачем нужен этот скрипт?

Ручная установка GLPI — это рутинный и долгий процесс:

    Нужно установить десяток PHP-модулей.

    Настроить MariaDB, загрузить в неё временные зоны.

    Разнести конфигурационные файлы и логи по разным папкам (в последних версиях GLPI это обязательное требование).

    Настроить права доступа (chown/chmod), крон-задачи и конфигурацию Apache.

Этот скрипт экономит ваше время и исключает ошибки, связанные с "забыл установить модуль" или "забыл выставить права на папку".
⚙️ Почему именно так? (Особенности архитектуры)

Скрипт написан не просто для того, чтобы "заработало", а так, как того требует официальная документация GLPI:

    Соответствие FHS (Filesystem Hierarchy Standard):

        Вместо хранения всего в /var/www/glpi, скрипт безопасно распределяет данные:

            Конфиги ➡️ /etc/glpi

            Загрузки и кеш ➡️ /var/lib/glpi/files

            Плагины ➡️ /var/lib/glpi/plugins

            Логи ➡️ /var/log/glpi

        Доступ к этим путям настраивается через файлы downstream.php и local_define.php.

    Безопасность (Hardening):

        Выполняется базовая настройка безопасности MariaDB (удаление анонимных пользователей и тестовых БД).

        Web-сервер Apache смотрит только в папку public, закрывая доступ к системным файлам извне.

        Строгое разделение прав: ядро GLPI принадлежит root, а веб-серверу www-data отдаются только те папки, куда ему нужно писать (marketplace, кеш, логи).

    Готовность к API и CalDAV:

        Apache настраивается с пробросом заголовка Authorization, что необходимо для корректной работы API и синхронизации календарей.

    Современный стек:

        Автоматически подключается репозиторий ondrej/php для установки PHP 8.2 (максимально поддерживаемая версия для GLPI 10.0.7).

🛠 Требования

    ОС: Ubuntu 20.04 / 22.04 / 24.04

    Права: root или пользователь с правами sudo

    Интернет: Наличие доступа в сеть (для скачивания пакетов и архива с GitHub)

    Диск: Минимум 1 ГБ свободного места

🚀 Как использовать?
1. Скачивание скрипта

Склонируйте репозиторий или просто скачайте скрипт:
code Bash

git clone https://github.com/ВАШ_НИК/glpi-autoinstall.git
cd glpi-autoinstall

(Или скачайте напрямую через wget)
code Bash

wget https://raw.githubusercontent.com/ВАШ_НИК/glpi-autoinstall/main/autoinstall-glpi.sh

2. Запуск (Интерактивный режим)

Сделайте скрипт исполняемым и запустите:
code Bash

chmod +x autoinstall-glpi.sh
sudo ./autoinstall-glpi.sh

В процессе установки скрипт попросит вас придумать и ввести:

    Пароль для root пользователя MariaDB.

    Пароль для пользователя БД glpi.

3. Запуск (Автоматический / Неинтерактивный режим)

Идеально подходит для развертывания через Ansible, Terraform или cloud-init. Передайте переменные окружения перед запуском:
code Bash

sudo MYSQL_ROOT_PASSWORD="SuperSecretRoot" \
     DB_PASSWORD="GlpiSecretPassword" \
     SERVER_NAME="helpdesk.mycompany.com" \
     GLPI_VERSION="10.0.7" \
     ./autoinstall-glpi.sh

Доступные переменные:
Переменная	По умолчанию	Описание
GLPI_VERSION	10.0.7	Версия GLPI для скачивания с GitHub.
SERVER_NAME	glpi.local	Доменное имя для конфигурации Apache (VirtualHost).
PHP_VERSION	8.2	Версия PHP. Внимание: GLPI 10.0.7 не поддерживает PHP 8.3+.
MYSQL_ROOT_PASSWORD	-	Пароль root от БД (если пустой — скрипт спросит).
DB_PASSWORD	-	Пароль пользователя glpi для БД (если пустой — скрипт спросит).
🏁 Что дальше (После установки)?

    Откройте в браузере адрес, который вы указали в SERVER_NAME (например, http://glpi.local или IP сервера).

    Выберите язык и запустите установку.

    На этапе настройки базы данных укажите:

        Сервер SQL: localhost

        Пользователь SQL: glpi

        Пароль SQL: Пароль, который вы ввели для БД при работе скрипта.

        База данных: Выберите glpi из списка.

    После завершения мастера установки, скрипт install/install.php будет удален автоматически самим GLPI.

Рекомендации по HTTPS

По умолчанию скрипт настраивает HTTP (порт 80). Рекомендуется установить SSL-сертификат:
code Bash

sudo apt install certbot python3-certbot-apache
sudo certbot --apache -d helpdesk.mycompany.com

После включения HTTPS рекомендуется раскомментировать строку session.cookie_secure = On в файле /etc/php/8.2/apache2/php.ini для повышения безопасности сессий.
📝 Логирование

Все действия скрипта записываются и выводятся в консоль в цветном виде.
В случае возникновения проблем, подробный лог можно найти здесь:

    Во время установки: /tmp/glpi-install.log

    После успешной установки: /var/log/glpi-install.log
