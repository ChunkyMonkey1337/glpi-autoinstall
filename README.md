# GLPI Auto-Installer for Ubuntu

Скрипт для полной автоматизации развертывания системы GLPI (IT Service Management / Helpdesk) на серверах под управлением Ubuntu. Скрипт полностью соответствует официальной документации и современным стандартам безопасности.

## Что это и зачем нужно?

Ручная установка GLPI — это долгий процесс: требуется установить десяток PHP-модулей, настроить MariaDB, загрузить временные зоны, разнести конфигурационные файлы и логи по разным папкам (обязательное требование новых версий), а также настроить права доступа и крон-задачи.

Этот bash-скрипт за несколько минут превращает "чистую" Ubuntu в готовый к работе сервер GLPI. Он экономит время и исключает ошибки, связанные с забытыми модулями или неверно выставленными правами на директории.

## Особенности архитектуры (Как это работает)

Скрипт написан в строгом соответствии с [официальной документацией GLPI](https://glpi-install.readthedocs.io/en/latest/install/index.html):

1. **Соответствие FHS (Filesystem Hierarchy Standard):**
   Вместо хранения всего в `/var/www/glpi`, скрипт безопасно распределяет данные по системе:
   - Конфиги: `/etc/glpi`
   - Загрузки и кеш: `/var/lib/glpi/files`
   - Плагины: `/var/lib/glpi/plugins`
   - Логи: `/var/log/glpi`
   Доступ к этим путям настраивается автоматически через файлы `downstream.php` и `local_define.php`.

2. **Безопасность (Hardening):**
   - Выполняется базовая настройка безопасности MariaDB (удаление анонимных пользователей и тестовых БД).
   - Web-сервер Apache смотрит **только** в папку `public`, закрывая доступ к системным файлам извне.
   - Строгое разделение прав: системные файлы GLPI принадлежат `root`, а веб-серверу `www-data` отдаются права на запись только туда, где это необходимо (marketplace, кеш, логи).

3. **Готовность к API и CalDAV:**
   Apache настраивается с пробросом заголовка `Authorization`, что необходимо для корректной работы API и синхронизации календарей.

4. **Современный стек:**
   Скрипт использует репозиторий `ondrej/php` для установки PHP 8.2 (GLPI 10.0.7 требует версию ниже 8.3).

## Требования

- **ОС:** Ubuntu 20.04 / 22.04 / 24.04
- **Права:** `root` или пользователь с правами `sudo`
- **Интернет:** Доступ в сеть для скачивания пакетов и архива с GitHub

## Установка и использование

Склонируйте репозиторий и сделайте скрипт исполняемым:

```bash
git clone https://github.com/ChunkyMonkey1337/glpi-autoinstall.git
cd glpi-autoinstall
chmod +x autoinstall-glpi.sh
```

Интерактивный режим

Просто запустите скрипт. В процессе он попросит вас придумать и ввести пароли для MariaDB:
```bash

sudo ./autoinstall-glpi.sh
```
Неинтерактивный режим (Unattended)

Идеально подходит для автоматического развертывания через Ansible, Terraform или bash-скрипты cloud-init. Передайте параметры через переменные окружения:
```bash

sudo MYSQL_ROOT_PASSWORD="SuperSecretRoot" \
     DB_PASSWORD="GlpiSecretPassword" \
     SERVER_NAME="helpdesk.mycompany.com" \
     GLPI_VERSION="10.0.7" \
     ./autoinstall-glpi.sh
```
Доступные переменные:
Переменная	По умолчанию	Описание
GLPI_VERSION	10.0.7	Версия GLPI для скачивания с GitHub.
SERVER_NAME	glpi.local	Доменное имя для конфигурации Apache (VirtualHost).
PHP_VERSION	8.2	Версия PHP.
MYSQL_ROOT_PASSWORD	пуст	Пароль root от БД (если не указан, скрипт спросит).
DB_PASSWORD	пуст	Пароль пользователя glpi для БД (если не указан, скрипт спросит).
Дальнейшие шаги (После установки)

    Откройте в браузере адрес, указанный в SERVER_NAME (например, http://glpi.local или IP сервера).

    Выберите язык и запустите установку GLPI.

    На этапе настройки базы данных укажите:

        Сервер SQL: localhost

        Пользователь SQL: glpi

        Пароль SQL: Пароль, который вы ввели для переменной DB_PASSWORD (или указали при запуске).

        База данных: Выберите glpi из списка.

    После завершения мастера установки файл install/install.php будет удален системой автоматически.

Рекомендации по HTTPS:
По умолчанию скрипт настраивает HTTP (порт 80). Настоятельно рекомендуется установить SSL-сертификат (например, через Let's Encrypt / Certbot). После включения HTTPS раскомментируйте или добавьте строку session.cookie_secure = On в файле /etc/php/8.2/apache2/php.ini для повышения безопасности сессий.
Логирование

В случае возникновения проблем, подробный лог установки можно найти здесь:

    Во время работы скрипта: /tmp/glpi-install.log

    После завершения установки: /var/log/glpi-install.log
